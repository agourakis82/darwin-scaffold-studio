# Add Feature

Plan and implement a new feature for Darwin Scaffold Studio.

## Arguments
$ARGUMENTS - Description of the feature to add

## Instructions

1. **Understand the request**
   - Parse the feature description
   - Identify which modules are affected

2. **Research existing code**
   - Find related functionality
   - Check coding patterns used
   - Identify dependencies needed

3. **Create implementation plan**
   - List files to modify/create
   - Define function signatures
   - Identify tests needed

4. **Implement step by step**
   - Follow existing code style
   - Add proper docstrings
   - Export public functions
   - Handle errors gracefully

5. **Test the feature**
   - Add unit tests
   - Run module loading test
   - Verify no regressions

6. **Document**
   - Update relevant docs
   - Add usage examples

## Code Style Rules
- Use descriptive function names
- Add docstrings with examples
- Type annotations for public functions
- Follow Julia naming conventions (snake_case for functions, PascalCase for types)
- Keep functions focused and small
